# changelog.nextcommerce.com → docs.nextcommerce.com/changelog

A Cloudflare Worker that 301-redirects every request on
`changelog.nextcommerce.com` to the equivalent location under
`https://docs.nextcommerce.com/changelog`.

## Why

The old changelog lived on a Next Commerce storefront Blog section
(`/blog/detail/{date-slug}/`) with 149 entries going back to May 2020. This
worker preserves every historical deep link: each old URL is mapped to its
new kebab-case slug in `slug-map.json`. Anything that doesn't match a known
detail path (index page, unknown slugs) falls back to the new `/changelog`
index.

## Files

| File | Purpose |
|------|---------|
| `worker.ts` | Matches `/blog/detail/{slug}/` and looks up `slug-map.json`. |
| `slug-map.json` | Generated by `tools/process-changelog.py`. Edit via that script. |
| `wrangler.jsonc` | Binds the worker to `changelog.nextcommerce.com`. |

The slug map is regenerated any time the processing pipeline is re-run (for
example, when importing further historical entries or renaming slugs). Commit
the regenerated file alongside the worker.

## Deploy

```sh
cd redirects/changelog-legacy
npx wrangler deploy
```

Before the first deploy, the Cloudflare dashboard route on
`changelog.nextcommerce.com` must be detached from whatever currently serves
it (the legacy NEXT store). The `custom_domain` entry in `wrangler.jsonc`
will attach it to this worker.

## Verify

```sh
# A known legacy slug should 301 to its new counterpart.
curl -sI https://changelog.nextcommerce.com/blog/detail/18032026/
# Expect: HTTP/2 301
# Expect: location: https://docs.nextcommerce.com/changelog/2026-03-20-subscriptions-payments-campaigns-checkout

# Unknown paths fall back to the index.
curl -sI https://changelog.nextcommerce.com/blog/
# Expect: HTTP/2 301
# Expect: location: https://docs.nextcommerce.com/changelog
```
