The Meta Pixel App creates a server-to-server connection to post tracking and sales attribution metrics to the (Facebook) Meta Conversions API.  Server-side API tracking is the method recommended by Meta to ensure the highest degree of accuracy in the data sent to the Meta Pixel.

<Callout type="info">

**Meta Pixel** is an installable App. Enable it from the **Apps** menu on your store.

</Callout>

## Set Up Meta Pixel

To connect your store to the Meta Conversions API, navigate to **Apps > Meta Pixel > Settings.**

Select **Add Pixel**

Input the following values obtained from your Meta Business Manager account:

* Pixel ID (Dataset ID)
* Access Token

Select **Save.**

You may optionally add additional Meta Pixels to your store.

### Meta Settings

Once you have added one or more Meta Pixels, choose the **Default Meta Pixel** to be used in your store.  Configure the following optional settings

* **Enable Delay Order Confirmation** - add a 10 minute delay after purchase events, to ensure that orders are completed including any upsells before being posted to the Conversions API.
* **Enable Test Data** - allow [**Test Orders**](/docs/manage/orders/test-orders) data to be posted to the Conversions API.

### How Meta Pixel App works

The **Meta Pixel** app automatically adds your Meta Pixel JS to the storefront.  Additionally, the app creates the following attribution [**metadata**](/docs/build-a-store/technical-settings/metadata-fields-and-tags#metadata) fields in your store, which correspond to values required by Meta Conversions API to track events accurately:

* **fb\_fbc** (required) - value from Meta Pixel JS to capture the click.
* **fb\_fbp** (required) - value from Meta Pixel JS to capture the click.
* **fb\_pixel\_id** (optional) - pass an alternative **Pixel ID** to specify a different Meta Pixel than the app's Default to be used in the Meta Conversions API post.
* **fb\_test\_code** (optional) - this field can be used to pass test events, but should only used by developers.  Additional documentation - [**Meta**](https://developers.facebook.com/docs/marketing-api/conversions-api/using-the-api/#testEvents).
* **fb\_event\_source\_url** (optional) - pass the url the event happened on such as an external marketing campaign.

<Callout type="info">

For additional documentation about Click IDs and related tracking parameters, see [**Meta documentation**](https://developers.facebook.com/docs/marketing-api/conversions-api/parameters/fbp-and-fbc)

</Callout>

## Use Meta Pixel with External Checkouts

When creating orders via externally-hosted marketing funnels and checkouts integrated to the [**Admin API**](https://developers.nextcommerce.com/docs/admin-api/guides/external-checkout) or the [**Campaign API**](https://developers.nextcommerce.com/docs/campaigns/api), it's important to pass the required metadata on orders to ensure that the Meta Conversions API receives the proper attribution elements.

* **fb\_fbc** (required)
* **fb\_fbp** (required)

These represent browser cookie values generated by the Meta Pixel JS, installed on your externally-hosted site.  Capturing these values and appending them to orders as [**attribution metadata**](https://developers.nextcommerce.com/docs/admin-api/guides/external-checkout#cart--order-attribution-detail), will post them to the Meta Conversions API on order events.
